# Cursor User Rules 用（追記：.gitignore）

**ブランチの切り方・コミットの進め方などは、もともとある User Rules に従う。** ここで新しく足すのは **`.gitignore` の運用**だけ。

このファイルの本文は、**Cursor の設定 → Rules（User Rules）** にコピーして使う想定です（既存のルールの**末尾に追記**でよい）。  
リポジトリには「貼り付け用の正本」としてだけ置く。`.cursor/rules` には同じ長文を複製しない。

---

## .gitignore の運用

- `.gitignore` に**不要な行を書かない**。無視すべき成果物が実際に現れたタイミングで、必要なパターンだけ足す。
- 「いつか使うかも」だけを理由に、長いテンプレートを丸ごと貼らない。
- **ビルド成果物・依存ディレクトリ・秘密情報ファイル**など、コミットしてはいけないものがプロジェクトに導入されたら、その都度 `.gitignore` に追記する。
- **OS やエディタ固有のゴミ**（例: `.DS_Store`）は、可能なら**利用者の Git グローバル exclude**で無視し、プロジェクトの `.gitignore` を薄く保つ。
- 無視対象がまだ無い段階では **`.gitignore` ファイル自体を置かない**選択も妥当。
