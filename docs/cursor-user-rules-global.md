# Cursor User Rules 用（グローバル）

このファイルの本文は、**Cursor の設定 → Rules（User Rules）** にコピーして使う想定です。  
リポジトリには「貼り付け用の正本」としてだけ置きます。`.cursor/rules` には同じ長文を複製しません。

---

## リポジトリと Git の運用

### 方針

- リポジトリには**必要なものだけ**置く。用途のないファイルや先回りの設定で汚さない。
- `.gitignore` にも**不要な行を書かない**。無視すべき成果物が実際に現れたタイミングで、必要なパターンだけ足す。
- 「いつか使うかも」だけを理由に、長いテンプレートを丸ごと貼らない。

### .gitignore

- **ビルド成果物・依存ディレクトリ・秘密情報ファイル**など、コミットしてはいけないものがプロジェクトに導入されたら、その都度 `.gitignore` に追記する。
- **OS やエディタ固有のゴミ**（例: `.DS_Store`）は、可能なら**利用者の Git グローバル exclude**で無視し、プロジェクトの `.gitignore` を薄く保つ。
- ドキュメントのみなど、無視対象がまだ無い段階では **`.gitignore` ファイル自体を置かない**選択も妥当。

### ブランチとコミット

- 作業内容ごとに**ブランチを切ってから**変更する（`main` への直接コミットは避ける）。
- コミット前に**コミットメッセージ案**を示し、ユーザーから許可を得てからコミットする。
- 許可後は**マージ・プッシュ**し、作業終了時は **`main` に戻る**。
