# MVP 動作確認チェックリスト（Phase 10）

この文書は [実装計画の Phase 10](implementation-plan.md) に対応する。一連の流れが手動で通るか、主要なエラー表示が確認できるかを自分で検証するときに使う。

## 前提（そろえておくもの）

- Google Cloud で **Speech-to-Text API** と **Google Docs API** が有効なプロジェクトがある。
- サービスアカウントの **JSON キー**があり、環境変数 `GOOGLE_APPLICATION_CREDENTIALS` にその**絶対パス**を指せる。
- 出力先の **Google ドキュメント**を、そのサービスアカウントのメールアドレスに **編集者として共有**している。
- このリポジトリの **relay** で `npm install` 済み、**Chrome** に **拡張機能**を読み込み済み（Developer mode）。

## 正常系（最初から最後まで）

次の順で進め、各項目を満たせば正常系は一通り通っている。

1. **relay を起動する**  
   `relay` ディレクトリで `npm start` を実行し、ターミナルに `WebSocket listening` が出ることを確認する。

2. **拡張機能を再読み込みする**（コードや manifest を変えた直後なら）  
   `chrome://extensions` で当該拡張の「再読み込み」を押す。

3. **Google Meet に参加したタブを手前にする**  
   別タブだけを前面にしたままだとタブ音声取得が拒否されることがある。

4. **拡張のポップアップを開き、出力先ドキュメントの URL を入力する**  
   ドキュメント ID がヒントに出る形式であればよい。

5. **記録開始を押す**（このとき **Meet のタブが手前**であること）  
   録音用のタブ（`mic.html`）が開く。

6. **録音タブを選び、「Meet とマイクを許可して録音開始」を押す**  
   中継へ接続し、Meet のタブ音声とマイクをミックスして送信が始まる。

7. **しばらく会話または再生する**  
   relay のログに `binary audio chunk` が出続けること。認証があれば `transcript` と Docs 追記のログが出ること。

8. **Google ドキュメントを開き、末尾に `[自動]` 付きの文が増えていくことを確認する**（認証と共有が正しければ）。必要ならドキュメントのタブを手前にして共同編集してよい。

9. **録音タブで記録停止を押す**  
   録音が終わり、タブが閉じる。ポップアップは「待機中」でよい。

## エラー時の表示（主なパターン）

手動で状況を作り、ポップアップまたは relay のログで次が分かることを確認する。

| 想定する状況 | 確認したいこと |
|--------------|----------------|
| relay を起動していない、またはポートが違う | 拡張のステータスに、中継へ接続できない旨のメッセージが出る |
| Meet を手前にせず記録開始した | ポップアップに Meet を手前にするよう促すメッセージが出る |
| `GOOGLE_APPLICATION_CREDENTIALS` が未設定 | relay 起動ログで STT と Docs がスキップされる旨が出る。文字と Docs は増えない |
| ドキュメントをサービスアカウントに共有していない | relay に Docs API のエラーが出る。ドキュメント本文は追記されない |

## MVP 完了の自己判定

[実装計画の「MVP 完了の定義」](implementation-plan.md) と [システム要件](system-requirements.md) に照らし、少なくとも次を満たせば MVP として一通り揃ったとみなしてよい。

- 会議中に、確定テキストがドキュメント末尾に追記される（環境が揃っている場合）。
- 人間が同じドキュメントで誤字などを直せる。
- システムは末尾への追記のみで、既存行を意図的に置換しない（設計どおり）。
